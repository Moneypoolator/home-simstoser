
# Анализ состояния проекта S3-хранилища

Проект представляет собой S3-совместимый сервер хранения файлов на C++20 с веб-интерфейсом. Архитектура модульная, используются современные библиотеки (Boost.Asio/Beast, OpenSSL, nlohmann/json, glog). Реализована аутентификация AWS Signature V4, ролевая авторизация, многопоточный асинхронный сервер, поддержка HTTPS, ограничение частоты запросов, multipart и stream upload.

## Задачи на реализацию

| Задача | Текущее состояние | Рекомендация |
|--------|-------------------|--------------|

| Загрузка/сохранение политик из файла | Политики хранятся только в памяти. Нет методов `load_policies`/`save_policies`. | Реализовать аналогично `load_users`/`load_acls`, добавить в authorizer и вызывать при старте/останове. |
| Проверка клиентских SSL-сертификатов | В `ssl_config` есть `verify_client`, но по умолчанию `false`. Нет тестов/примеров использования. | Протестировать сценарий с взаимной TLS-аутентификацией, добавить документацию. |

## 💡 Предложения по дальнейшему развитию

1. **Асинхронная запись в журнал аудита**

Сейчас используется `glog`, который синхронно пишет в файл. Для production можно добавить асинхронный логгер (например, `spdlog`) с ротацией и записью структурированных событий (JSON) для последующего анализа.

2. **Улучшенная обработка ошибок и retry для multipart upload**

Клиенты могут повторять загрузку частей при сбоях. Сейчас сервер перезаписывает существующую часть. Стоит добавить проверку `ETag` части при повторной загрузке (сравнение с ранее загруженной).

3. **Поддержка S3 Event Notifications**

Отправка уведомлений (webhook, AMQP, Kafka) при создании/удалении объектов. Полезно для интеграции с внешними системами.

4. **Инкрементальное резервное копирование метаданных**

При большом количестве файлов `list_files` обходит всю ФС. Можно хранить индекс в SQLite для быстрого поиска и листинга.

5. **Горячая перезагрузка конфигурации**

Отслеживание изменений JSON-файла и применение новых настроек (например, лимитов, CORS) без перезапуска сервера.

6. **Поддержка S3 Transfer Acceleration**

Использование нескольких потоков для загрузки/скачивания больших файлов (клиентская часть, но сервер может поддерживать range-запросы с параллельным чтением).

7. **Встроенный веб-интерфейс администратора**

Расширить существующий React-фронтенд страницами управления пользователями, политиками, просмотром статистики.

8. **Интеграция с OAuth2/OIDC провайдерами**

Дополнить `authenticator` возможностью валидации JWT токенов, выпущенных Keycloak, Auth0 и т.д.

### Повышение качества тестирования

1. Мокирование зависимостей: Для юнит-тестов request_handler стоит использовать моки file_manager, authenticator, authorizer, чтобы изолировать логику обработки HTTP.

1. Тесты производительности: Добавить бенчмарки для критических операций (подпись, хеширование, кэширование).

1. Fuzz-тестирование: Для парсеров (Authorization header, query string) полезно подавать случайные некорректные данные и проверять отсутствие крашей.

1. Тесты на утечки памяти: Использовать AddressSanitizer/LeakSanitizer при прогоне тестов.
